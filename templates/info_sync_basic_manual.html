info_sync 매뉴얼

작품 폴더를 스캔해 info.xml / cover 현황을 확인·편집하고, 여러 사이트에서 메타데이터를 가져와 자동으로 채워 넣는 플러그인.

1. 사용 순서
  1. 설정 탭에서 스캔 루트 폴더를 지정한다(작품 폴더들이 들어있는 상위 폴더).
  2. Info 관리 탭 → 스캔: 폴더 안 각 작품의 info.xml·cover 유무를 표로 본다.
  3. 매칭 스캔: 폴더명을 제목으로 전 사이트를 교차조회해 후보를 찾는다. 제목 정규화 완전일치가 1곳이면 확정, 여러 곳이면 애매(대표 소스 우선순위로 기본 선택)로 분류된다.
  4. 후보를 확인·선택한 뒤 info 작성 / cover 작성을 실행한다. info 작성 시 표지가 없으면 함께 내려받는다.
2. 설정 항목
  • 스캔 루트 폴더 — 작품 폴더들의 부모 경로.
  • 대표 소스 우선순위 — 완전일치가 여러 사이트일 때 앞쪽을 대표로 삼는다(콤마 구분).
  • 빈 필드 병합 채우기 — 켜면 대표 소스의 빈 필드만 다른 사이트 완전일치 후보에서 채운다(기존 값은 덮지 않음).
  • cover 덮어쓰기 — 끄면 표지가 없을 때만 저장, 켜면 항상 새로 저장.
  • 사이트별 쿠키 — 대부분 조회엔 불필요. 성인/로그인 필요한 작품만 JSON 배열 형식으로 입력.
3. 지원 사이트 (10종)

설정 탭의 provider 확인 버튼으로 현재 사용 가능 여부를 볼 수 있다.

  • 웹툰/만화 — 카카오웹툰, 네이버웹툰, 레진코믹스, 투믹스
  • 웹소설 — 카카오페이지, 네이버시리즈, 노벨피아, 문피아, 조아라
  • 도서/전자책 — 리디북스(웹툰·웹소설·책)
4. 리디북스 조회 준비 — flaresolverr 필요

리디북스(ridibooks.com)는 전 페이지가 Cloudflare로 보호돼 일반 요청은 전부 차단된다(403). 그래서 리디만은 flaresolverr(실제 브라우저로 Cloudflare를 통과시켜 주는 프록시)가 별도로 설치·구동되어 있어야 한다. 나머지 9개 사이트는 flaresolverr 없이 동작한다.

① flaresolverr 컨테이너 설치

도커가 있는 환경에서 아래처럼 띄운다(포트 8191).

docker run -d \
  --name flaresolverr \
  --restart unless-stopped \
  -p 8191:8191 \
  -e LOG_LEVEL=info \
  ghcr.io/flaresolverr/flaresolverr:latest

시놀로지는 Container Manager에서 이미지 ghcr.io/flaresolverr/flaresolverr:latest를 받아 로컬 포트 8191로 매핑하면 된다.

② 설정 탭에 flaresolverr URL 입력

설정 탭의 flaresolverr URL 칸에 이 플러그인(FlaskFarm)이 도는 컨테이너에서 flaresolverr에 닿는 주소를 넣는다. 끝에 /v1을 붙인다.

  • 두 컨테이너가 기본 bridge에 있으면 → http://172.17.0.1:8191/v1 (도커 호스트 게이트웨이. 기본값으로 이미 넣어져 있음)
  • 둘이 같은 사용자 정의 네트워크에 있으면 → http://flaresolverr:8191/v1 (컨테이너 이름)
  • FlaskFarm이 컨테이너가 아닌 호스트에서 직접 돌면 → http://localhost:8191/v1

확인: 브라우저나 curl http://<host>:8191/에서 flaresolverr 상태 메시지가 나오면 정상. 리디 성인물(19)은 리디북스 쿠키도 함께 입력해야 조회된다(쿠키는 flaresolverr로 전달됨).

5. 참고
  • info.xml은 ComicInfo 형식으로 작성되며, 어느 사이트에서 가져왔는지 MetaSource에 기록된다.
  • 표지는 cover.jpg(또는 유사 파일명)로 작품 폴더에 저장된다.
  • 대부분의 조회는 로그인 없이 되지만, 성인물·유료작은 해당 사이트 쿠키가 필요할 수 있다.
